For Java Developers

Multimap comes with a language binding for Java based on JNI. Although it's Java, the target platform of the native library is still GNU/Linux. At this time, the library has been tested under Debian 7.8 (wheezy) and 8.1 (jessie) both for x86-64 and x86-32.

The following steps will build

  1. a shared library with JNI support,
  2. a JAR file to be added to your project's classpath

Download Release Tarball

from GitHub: https://github.com/mtrenkmann/multimap/releases

cd path/to/download
tar xzvf multimap-{version}.tar.gz
cd multimap-{version}

Install Build Tools

Multimap uses qmake as its Makefile generator.

sudo apt-get install qt5-make
# For Debian 8 based distros

sudo apt-get install qt4-make
# For Debian 7 based distros

# Don't know your Debian release?
# lsb_release -r

sudo apt-get install make g++

Install Dependencies

sudo apt-get install libboost-filesystem-dev libboost-system-dev libboost-thread-dev

Define JAVA_HOME

In order to build the native library with JNI support, some header files from a Java Development Kit (JDK) are needed. If not yet available, install a JDK and define the JAVA_HOME environment variable pointing to it.

sudo apt-get install openjdk-7-jdk

export JAVA_HOME=/usr/lib/jvm/java-7-openjdk-amd64
# For Debian 8 based distros (x86-64)

export JAVA_HOME=/usr/lib/jvm/java-7-openjdk-i386
# For Debian 8 based distros (x86-32)

Build JNI Shared Library

qmake multimap-library-jni.pro
make

# Sometimes you need to run qmake directly from its installation directory
# Debian 8 (x86-64): /usr/lib/x86_64-linux-gnu/qt5/bin/qmake
# Debian 8 (x86-32): /usr/lib/i386-linux-gnu/qt5/bin/qmake
# Debian 7 (x86-64): /usr/lib/x86_64-linux-gnu/qt4/bin/qmake
# Debian 7 (x86-32): /usr/lib/i386-linux-gnu/qt4/bin/qmake

Install JNI Shared Library

sudo make install

This will install the shared library libmultimap-jni.so in /usr/local/lib. If you don't have permission to install the library system-wide, you can copy it somewhere else and let the Java library path pointing to that location. For example, if the native library is in /home/user/downloads/multimap-{version} you can add

-Djava.library.path=/home/user/downloads/multimap-{version}

to the VM arguments of your Java project.

Build Java Library

./make-multimap-jar.sh

This builds the JAR file multimap-{version}.jar that you can add to your project's classpath.
